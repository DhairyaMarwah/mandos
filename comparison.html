01 · Where you are

Your stack today. Honest read first.

Mandos runs Express + AWS Lambda + S3 + Socket.io + n8n + RDS Postgres. It works. 1,500 users, no marketing — the stack isn't blocking anything right now. The two parts that will hurt at scale are S3 egress on media and RDS over-provisioning. Everything else is fine for the moment.

Keep as-is
Postgres + Prisma + TS. Boring, correct.
Stripe · Twilio · SendGrid · Firebase push. Right primitives.
Sentry + Jest. Real observability + test runner.
Lambda for sporadic endpoints. Pay-per-request makes sense there.
Worth swapping eventually
S3 egress at $0.09/GB. Largest growing line item.
RDS over-provisioning. $150-500/mo for 1,500 users. Wildly oversized.
Socket.io single-node ceiling. ~10K concurrent before sharding work.
n8n as production infra. Workflow tools become tech debt.
02 · The four cost killers

Where the money leaks. In order of leverage.

Four things on the current stack cost meaningfully more than they need to. Each has a low-effort swap. The other 80% of the stack stays exactly as it is.

01 · highest leverage

S3 egress

$0.09/GB on outbound media. Biggest line item past 50K MAU. R2 has zero egress.

02 · second biggest

RDS Postgres

db.t3.medium with multi-AZ + backups runs $200-500/mo. Neon Launch is $19.

03 · structural ceiling

Socket.io single-node

Caps around 10K concurrent. Sharding it is 2 weeks of work. Pusher is a drop-in.

04 · slow death

Lambda cold starts

~800ms on first hit. Hurts perceived speed. Only swap when growth justifies it.

03 · The path forward

Phased migration. No big-bang rewrite.

Three phases, smallest blast radius first. Phase 1 is a week of work and changes zero application code — just bucket URLs. Phase 3 is "only when growth justifies it." The team keeps shipping product throughout.

Phase 1 · This week
Easy wins.
Two swaps. Zero application-code changes. Real money saved on day one.
AWS S3 → Cloudflare R2src ↗
S3-compatible API. Point the bucket URLs at R2, swap credentials. Existing aws-sdk code works unchanged. Then turn off S3 egress.
Saves $1,547/mo
Effort ~1 day
n8n → Inngestsrc ↗
Rewrite the few real workflows as Inngest functions. Drop the n8n VM. Code-defined, versioned in git, retries baked in.
Saves ~$50/mo
Effort ~1 week
Phase 2 · This month
High leverage.
Two swaps that change the structural ceiling. Both are drop-in: same Prisma, same client APIs.
RDS Postgres → Neonsrc ↗
Migrate via pg_dump. Same Prisma schema, same SQL. Get serverless autoscale + branching for previews. Massive cost drop.
Saves $180/mo
Effort ~3 days
Socket.io → Pusher Channelssrc ↗
Pusher exposes a Socket.io-compatible client surface. Server SDK swap, channel naming swap. No sharding to maintain ever again.
Saves $300-$2K/mo at scale
Effort ~1 week
Phase 3 · Only when growth justifies
Hot-path moves.
Only when the cold start tax is actually hurting users. Not Week 1. Maybe Q3.
Lambda → Vercel Fluidsrc ↗
For hot endpoints where cold starts hurt UX. Leave low-traffic endpoints on Lambda — pay-per-request is still cheap there.
Wins no cold start
Effort ~2-3 weeks
Express → Next.js + tRPC
Only for new endpoints. Get end-to-end types. Don't rewrite anything that already works.
Wins type-safety
Effort incremental

Biggest single line item

S3 egress at $0.09/GB → R2 at $0/GB.

With video clips and venue photos at 8MB+ per session, you're sending 17,187 GB per month. On S3 that's $1,547/mo. On R2 that's $0/mo egress. Same API. Same Prisma. The change is in your bucket URL only.

Second biggest line item

RDS Postgres → three serverless alternatives.

RDS db.t3.medium runs ~$50/mo base, but multi-AZ + automated backups + 100 GB GP3 storage easily lands at $200-$500/mosrc ↗. For 1,500 users this is wildly over-provisioned. Three Postgres alternatives that keep Prisma + your existing schema intact:

Neonsrc ↗
$19 / mo Launch tier
Serverless. Auto-pause when idle. Branching for previews. Strongest pick for Mandos's stage.
Supabasesrc ↗
$25 / mo Pro tier
Postgres + bundled Auth + Storage + Realtime. Heavier platform but consolidates services.
DigitalOcean Managedsrc ↗
$15 / mo · 1 GB tier
Predictable flat pricing, no surprise bills. Cheapest if you want plain managed Postgres without serverless magic.
05 · Real-time

Three drop-in WebSocket options. All keep Postgres + Prisma.

When Socket.io's single-node ceiling actually hurts — when you're seeing dropped messages at Friday-8pm — these three are drop-in replacements. Pick by how managed you want it.

01 · fastest to ship

Pusher Channelssrc ↗

$49/mo Pro · 200K daily msgs · 500 concurrent

Managed WebSockets. Server SDK swap, channel naming swap. Same Socket.io-like client surface. Recommended pick.

02 · generous free tier

Ablysrc ↗

Free up to 6M msgs/mo · scales

Alternative to Pusher. Bigger free tier, scale-grade SLAs. Worth comparing if you want a longer free runway.

03 · cheapest at scale

Cloudflare Durable Objectssrc ↗

$0.15/M reqs + $12.50/M GB-s

One Object per venue. Auto-shards globally. Cheapest above 100K concurrent but heavier onboarding.
