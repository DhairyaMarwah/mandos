An honest comparison of the current Mandos stack (Express on Node.js, AWS Lambda + S3, Socket.io, RDS Postgres, n8n) against a recommended hybrid for the next 100K MAU. Every cost claim links to the vendor's pricing page. Move the sliders, verify the numbers.

Your use case first.Then the recommendation.

Mandos is a Hong Kong nightlife app with real-time presence (live map showing who's out), media-heavy content (event posters, venue photos, user receipts), transactional bookings via Stripe, and a 1,500-user base targeting 100K+ MAU in the next 12 months. The infra has to handle Friday-8pm spikes without dropping connections.

What's working today
Postgres + Prisma + TypeScript. The boring, correct choice. Keep all three.
Stripe, Twilio, SendGrid, Firebase push. Right primitives for each job. Don't touch.
Sentry + Jest. Real observability + a real test runner. Most early-stage apps don't have this.
The product ships. 1,500 users with no marketing means the stack isn't blocking growth right now.
AWS Lambda is fine for sporadic workloads. If endpoints are infrequent, Lambda's pay-per-request makes sense.
Where it will hurt at scale
S3 egress at $0.09/GBsrc ↗. Mandos is media-heavy. This becomes the biggest line item past 50K MAU.
Lambda cold starts. 500-1500ms on first hit. Painful for a snappy social product where every tap should feel instant.
Socket.io on a single Node process. Caps around 10K concurrent connections. Friday-8pm spikes will hit this fast.
n8n as production infra. Workflow tools become tech debt. Code-defined workflows scale better and version with the app.
Express + ad-hoc API layout. No type-safe client-server contract. Slows feature velocity as the team grows.
Current · what you're running today
AWS Lambda · Express · S3 · n8n
Solid, conventional. Works for what it's serving today. Two pieces will become the bottleneck at scale: S3 egress on media, and Socket.io on a single Node host.
vs
Recommended · the hybrid we'd build for the next stage
Vercel · Cloudflare R2 · Pusher · Neon
Vercel + Fluid Compute for DX and cold-start fix. Cloudflare R2 for zero-egress media. Pusher (or Durable Objects) for real-time. Neon for serverless Postgres. Migration is component-by-component, no rewrite.
