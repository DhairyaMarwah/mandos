Cited pricing

Your use case first.Then the recommendation.

Mandos is a Hong Kong nightlife app with real-time presence, media-heavy content (event posters, venue photos, user-uploaded clips, soon video previews), transactional bookings via Stripe, and a 1,500-user base targeting 100K+ MAU. The infra has to handle Friday-8pm spikes without dropping connections.

What's working today
Postgres + Prisma + TypeScript. The boring, correct choice. Keep all three.
Stripe, Twilio, SendGrid, Firebase push. Right primitives for each job.
Sentry + Jest. Real observability + a real test runner. Most early apps don't have either.
The product ships. 1,500 users with no marketing means the stack isn't blocking growth right now.
AWS Lambda is fine for sporadic workloads. Pay-per-request makes sense for low-traffic endpoints.
Where it will hurt at scale
S3 egress at $0.09/GBsrc ↗. With video clips and venue photos at 5-30 MB per session, this becomes your biggest line item past 50K MAU.
Lambda cold starts. 500-1500ms on first hit. Painful for a snappy social product where every tap should feel instant.
Socket.io on a single Node process. Caps around 10K concurrent connections. Friday-8pm spikes will hit this fast.
n8n as production infra. Workflow tools become tech debt. Code-defined workflows scale better.
Express + ad-hoc API layout. No type-safe client-server contract. Slows feature velocity as the team grows.
Current · what you're running today
AWS Lambda · Express · S3 · Socket.io · n8n
Solid, conventional. Works for what it's serving today. Two pieces will become the bottleneck at scale: S3 egress on media, and Socket.io on a single Node host.
vs
Recommended hybrid · the next stage
Vercel Fluid · R2 · Convex / Pusher · Neon
Vercel + Fluid Compute for DX and zero cold start. Cloudflare R2 for zero-egress media (the biggest single win). Convex or Pusher for real-time. Neon for serverless Postgres. Migration is component-by-component.

Alternative · all-in-one path

Or consolidate with Convex.

Convexsrc ↗ is a reactive backend with type-safe functions, real-time queries, database, and file storage in one. For Mandos's real-time presence use case, Convex's reactivity model is purpose-built — every query auto-subscribes to changes and pushes updates to clients. Replaces Lambda + Postgres + Pusher + part of S3 in one platform.

Tradeoff: means leaving Prisma + Postgres for Convex's own DB. Heavier migration than a per-component swap. Best as a clean-slate or new-feature path, not a rip-and-replace.

The biggest single line item

S3 egress at $0.09/GB → R2 at $0/GB.

Mandos serves event posters, venue photos, user clips, and (soon) short video previews. At your current slider position that's 17,187 GB of egress per month. On S3 that's $1,547/mo. On R2 that's $0/mo egress. Storage cost is comparable. The egress is the savings.
